Maths · 01

Scalars, vectors & matrices

Intuition

A vector is a point in space. A 768-dim embedding is a point in 768D space where similar words are physically close together. The maths is identical to 3D — just more axes.

Shape notation — always read left to right

[3]→Vector — 3 numbers

Maths · 02

Dot product

a · b = a₁×b₁ + a₂×b₂ + ... + aₙ×bₙExample: [1,2,3] · [4,5,6] = 4+10+18 = 32 (scalar result)
It measures similarity — used everywhere in attention

Same direction → large positive. Perpendicular → 0. Opposite → negative. Attention computes Q·K = "how relevant is this key to this query?" — a dot product.

Week 2 + Week 3/4 connection

Perceptron: z = w·x + b (dot product). Cosine similarity in embeddings: sim(a,b) = (a·b)/(|a||b|). Attention scores: QKᵀ (many dot products at once). Same operation, different contexts.

Maths · 03

Matrix multiply

(m × n) @ (n × p) → (m × p)Inner dims cancel. Outer dims form the result. The core operation of every layer.
The forward pass is just this — and attention is too

Neural net layer: output = X @ W + b. Attention: Q = X @ W_Q, K = X @ W_K, scores = Q @ K.T. Every major operation in transformers is a matrix multiply.

XOR shape trace

[4,2]X @ W_ih [2,4]→ [4,4] hidden pre-activations
[4,4]a_h @ W_ho [4,1]→ [4,1] output predictions

Attention shape trace

[seq,d]X @ W_Q [d,d_k]→ [seq,d_k] Queries
[seq,d_k]Q @ K.T [d_k,seq]→ [seq,seq] attention scores
[seq,seq]weights @ V [seq,d_v]→ [seq,d_v] context-aware output
Maths · 04

Functions & derivatives

f(x) = x² → f'(x) = 2xSlope at x=3 is 6. A small nudge to x causes a 6× change in f(x).
FunctionDerivativeWhere it appears
f(x) = x²2xMSE gradient ∂L/∂ŷ = 2(ŷ−y)/n
f(x) = eˣeˣSigmoid, softmax, positional encoding

Sigmoid derivative: σ'(z) = σ(z) × (1 − σ(z)) = a(1 − a)Vanishes when a≈0 or a≈1. This is the vanishing gradient problem in deep sigmoid networks.
Maths · 05

Chain rule — the heart of backprop

If z = f(g(x)), then dz/dx = f'(g(x)) × g'(x)"Derivative of outer × derivative of inner." Backprop applies this backwards through every layer.
Week 2 concrete example

δ_output = (ŷ−y) × σ'(z_o) groups two chain rule steps. Then δ_hidden = (δ_output @ W_ho.T) × σ'(z_h) applies it again. loss.backward() does this automatically for all parameters.

Maths · 06

Gradients & gradient descent

w ← w − η × ∂L/∂wMove opposite to gradient. η = learning rate. Repeat millions of times = training.
OptimizerKey ideaWhen to use
SGDFixed η for all weightsSimple baselines, XOR
AdamAdaptive η per parameterMost deep learning
AdamWAdam + weight decayTransformers, LLMs, MiniLLM
NaN = learning rate too high. Plateau = too low.

XOR converges at η=2.0. MiniLLM uses 3e-4. Always start with a known-good value for the architecture type.

Maths · 07

Loss functions

MSE = (1/n) × Σ(yᵢ − ŷᵢ)²For regression and simple classification (XOR). Always positive, differentiable.
Cross-entropy = −log(probability of correct token)For LLMs. MiniLLM starts at ln(65) ≈ 4.17. Perfect = 0.
Why cross-entropy for LLMs

−log(0.001)=6.9 vs MSE (0−0.001)²=0.000001. Cross-entropy gives massively stronger gradient signal when the model is very wrong — essential for learning over 32K+ vocabulary.

In 1969, Minsky & Papert proved a single neuron cannot solve XOR. This triggered the first AI Winter. The solution: hidden layers transform the input space.

Neural Nets · 04

Forward pass

def forward(X):
    z_h = np.dot(X, W_ih) + b_h       # [4,2]@[2,4]=[4,4]
    a_h = sigmoid(z_h)                 # [4,4]
    z_o = np.dot(a_h, W_ho) + b_o      # [4,4]@[4,1]=[4,1]
    a_o = sigmoid(z_o)                 # [4,1] — predictions
    return z_h, a_h, z_o, a_o
Before training

Input [1,0] → random weights → ŷ = 0.46. Should be 1.0. That's fine — backprop will correct it over 10,000 steps. The forward pass is just prediction — learning happens in backward.

Neural Nets · 05

Loss & backpropagation

def backward(X, y, z_h, a_h, z_o, y_hat, lr):
    # Output layer — chain rule step 1
    delta_o = (y_hat - y) * sigmoid_derivative(y_hat)  # [4,1]
    dW_ho   = np.dot(a_h.T, delta_o) / 4                # [4,1] = ∂L/∂W_ho
    # Hidden layer — chain rule step 2 (error flows back through W_ho)
    delta_h = np.dot(delta_o, W_ho.T) * sigmoid_derivative(a_h)  # [4,4]
    dW_ih   = np.dot(X.T, delta_h) / 4                  # [2,4] = ∂L/∂W_ih
    # Gradient descent update
    W_ho -= lr*dW_ho;  W_ih -= lr*dW_ih
Why output first?

delta_h needs delta_o from the output layer. Error signal must flow backwards — output→hidden. Hence "backpropagation."

Text→Attention · 01

Tokenization — the model's "eyes"

Neural networks only understand numbers. Tokenization converts human-readable text into token IDs — the very first step before any learning can happen.

Three approaches — finding "just right"

ApproachExampleProblemVerdict
Character-level"Hi" → ['H','i'] → [72,105]Sequences too long, hard to learn meaningToo granular
Word-level"Hello" → [1234]Vocabulary explodes (millions), can't handle new wordsToo many
Subword (BPE)"unhappy" → ['un','happy']Token boundaries sometimes oddJust right ✅
BPE = Byte Pair Encoding — how modern LLMs tokenize

Starts with characters, then iteratively merges the most frequent adjacent pairs. Learns that "un" is a common prefix, "ing" a common suffix. GPT-4 uses cl100k_base with ~100K tokens vocabulary.

import tiktoken

# Basic usage

# Decode back

The strawberry problem — why LLMs can't count letters

"strawberry" → ['st','raw','berry']. The model never sees individual 'r' characters! It sees 3 tokens. This explains why LLMs struggle with letter-counting, reversing words, and spelling tasks — the atomic unit is the subword, not the character.

GPT-4's 128K limit is tokens, not words. JSON is nearly 2× less efficient than prose — colons, brackets, and quotes each use tokens. Keep prompts concise. "Summarize this" (2 tokens) costs 4× less than "Please provide a comprehensive summary of the following" (8 tokens) for the same instruction.

Text→Attention · 02

Embeddings — meaning in vectors

After tokenization we have IDs like [15496, 995]. These numbers are arbitrary — 15496 doesn't "know" it's a greeting. Embeddings transform these arbitrary integers into dense vectors where the numbers encode semantic meaning.

The goal: proximity = similarity

"Hello" and "Hi" should be mathematically close. "Hello" and "Banana" should be far apart. These relationships are learned — no one manually labels them. The model discovers semantic structure by processing billions of words.

The embedding matrix — a lookup table

import numpy as np
vocab_size    = 50000   # number of tokens
embedding_dim = 768     # GPT-2 style

# Learned during training — starts random, becomes meaningful
embedding_matrix = np.random.randn(vocab_size, embedding_dim) * 0.02
# Shape: [50000, 768] — 50K rows, one per token; 768 cols = feature dimensions
# Memory: ~153.6 MB (float32)

# Lookup — O(1) operation, just indexing a row
token_id  = 15496   # "Hello"
embedding = embedding_matrix[token_id]   # shape [768]
# This vector is the token's semantic fingerprint
Connection to nn.Embedding in PyTorch

torch.nn.Embedding(vocab_size, d_model) is this exact matrix. token_id → embedding is just matrix row lookup. In MiniLLM: nn.Embedding(65, 256) = 65-token vocab, 256-dim embeddings.

Semantic vector arithmetic

Embeddings are bag-of-words by default

They represent isolated meaning but ignore word order. "dog bites man" and "man bites dog" have the same set of embeddings. This is why we need positional encoding next.

Text→Attention · 03

Positional encoding — solving the order problem

Attention is permutation-invariant — it doesn't care about order. "The dog bit the man" and "The man bit the dog" are identical to a pure attention mechanism. We must inject position information.

# Proof that attention sees the same token set regardless of order
tokens1 = set(tokenizer.encode("dog bites man"))   # {18964, 66815, 893}
tokens2 = set(tokenizer.encode("man bites dog"))   # {893, 66815, 18964}
print(tokens1 == tokens2)  # True — identical set!

The solution — add a positional vector to each embedding

Final Input = Token Embedding + Positional EncodingEach word's vector now carries both semantic meaning AND its position in the sequence.

Sinusoidal positional encoding formula

PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))pos = position in sequence. i = dimension index. Different frequencies for different dimensions.
def get_positional_encoding(seq_length, d_model):
    position = np.arange(seq_length)[:, np.newaxis]      # (seq, 1)
    div_term = np.exp(np.arange(0, d_model, 2) * -(np.log(10000.0) / d_model))

    pe = np.zeros((seq_length, d_model))
    pe[:, 0::2] = np.sin(position * div_term)   # even dims: sine
    pe[:, 1::2] = np.cos(position * div_term)   # odd dims: cosine
    return pe   # shape (seq_length, d_model)

# Combine embedding + positional encoding
embeddings = np.random.randn(3, 8) * 0.1   # "The cat sat" — shape (3, 8)
pe         = get_positional_encoding(3, 8)
final_input = embeddings + pe                # shape (3, 8) — now position-aware

Why sine waves?

PropertyBenefit
Relative distancePosition p+k can always be expressed as a linear function of position p — model learns relative offsets naturally
Bounded [−1,1]Values stay stable during training — no explosion
No learned parametersFixed and pre-calculated — zero extra training cost
Generalises to longer sequencesWaves are continuous — model can extrapolate beyond training length
Modern LLMs use RoPE instead

Sinusoidal adds position to embeddings. RoPE (Rotary Position Embedding, used in MiniLLM, LLaMA, Mistral) rotates Q and K vectors by position angle. RoPE encodes relative distance, extrapolates better, and enables longer contexts. Both solve the same problem, RoPE solves it better at scale.

Text→Attention · 04

Self-attention — the magic of transformers

Self-attention lets every token in a sequence "look at" every other token simultaneously, creating context-aware representations. This is what makes transformers fundamentally different from RNNs.

Traditional RNNTransformer (attention)
Processes tokens one-by-oneProcesses all tokens simultaneously
Information passes through a bottleneckEvery token sees every other token directly
Long-range dependencies are hardLong-range dependencies are easy
Sequential — slowParallel — fast

Q, K, V — the three roles of every word

RoleQuestion it asksExample for "sat"
Query (Q)"What am I looking for?""Who sat? What did they sit on?"
Key (K)"What do I contain?""cat" says: "I'm an animal, a subject"
Value (V)"What do I offer if you pick me?""cat"'s actual semantic content
Library analogy

Q = your search terms. K = the index/catalog. V = the actual book content. Attention is a soft lookup: instead of returning one exact match, it returns a weighted mixture of all books proportional to how well each matches your search.

The attention formula

Attention(Q, K, V) = softmax( QKᵀ / √d_k ) × VThis single formula is the core of modern AI. Q:[seq,d_k], K:[seq,d_k], V:[seq,d_v] → output:[seq,d_v]

Step-by-step implementation

import numpy as np

def softmax(x):
    exp_x = np.exp(x - np.max(x, axis=-1, keepdims=True))  # stable softmax
    return exp_x / np.sum(exp_x, axis=-1, keepdims=True)

def self_attention(X, W_Q, W_K, W_V):
    # Step 1: Project to Q, K, V
    Q = X @ W_Q    # [seq, d_model] @ [d_model, d_k] = [seq, d_k]
    K = X @ W_K    # [seq, d_k]
    V = X @ W_V    # [seq, d_v]

    # Step 2: Compute attention scores
    d_k     = Q.shape[-1]
    scores  = Q @ K.T / np.sqrt(d_k)   # [seq, seq]

    # Step 3: Convert to probabilities
    weights = softmax(scores)           # [seq, seq] — rows sum to 1

    # Step 4: Weighted mix of values
    output  = weights @ V               # [seq, seq] @ [seq, d_v] = [seq, d_v]
    return output, weights

# Test on "The cat sat" with d_model=8, d_k=4
np.random.seed(42)
X   = np.random.randn(3, 8)
W_Q = np.random.randn(8, 4) * 0.1
W_K = np.random.randn(8, 4) * 0.1
W_V = np.random.randn(8, 4) * 0.1

output, weights = self_attention(X, W_Q, W_K, W_V)
print(f"Input:  {X.shape}")       # (3, 8)
print(f"Output: {output.shape}")  # (3, 4) — each word now context-aware
print(f"Weights: {weights.shape}") # (3, 3) — attention matrix

Reading the attention matrix

 The  cat  sat

Why "it" attends to "cat" not "street"

"The animal didn't cross the street because it was too tired." "it" sends a Query: "What noun am I referring to?" "animal" has a Key that matches (animals get tired, streets don't). "it"'s representation gets updated with "animal"'s Value features — disambiguation complete. This is coreference resolution.

Why scale by √d_k?

Without scaling: softmax saturation

Q and K both have unit variance. Their dot product QKᵀ has variance d_k. At d_k=64, raw scores are large → softmax assigns all probability to one token → gradient ≈ 0 → model stops learning. Dividing by √d_k brings variance back to 1, keeping softmax sensitive.

Text→Attention · 05

KV cache — the generation optimisation

During autoregressive generation (producing one token at a time), naively recomputing K and V for all previous tokens at every step is wasteful. The KV cache stores them.

The waste without cache

# Generating "cat" after "The":
#   Compute Q_the, K_the, V_the ← needed
#   Compute Q_cat, K_cat, V_cat ← needed (new)
#
# Generating "sat" after "The cat":
#   Compute Q_the, K_the, V_the ← SAME AS BEFORE (wasted!)
#   Compute Q_cat, K_cat, V_cat ← SAME AS BEFORE (wasted!)
#   Compute Q_sat, K_sat, V_sat ← needed (new)
#
# Only Q for the new token changes at each step.
# K and V for old tokens NEVER change.

With KV cache

# Step 1: "The"     → compute K_the, V_the → CACHE them
# Step 2: "cat"     → compute K_cat, V_cat → CACHE them; use cached K/V for "The"
# Step 3: "sat"     → compute K_sat, V_sat → CACHE them; use cached K/V for all
#
# Each step: only NEW token needs K,V computation.
# Everything else: look up cache → O(1)
Why first token is slow, rest are fast

First token: process ENTIRE prompt, compute K,V for every prompt token, fill the KV cache. Subsequent tokens: K,V already cached, only compute for new token. "Time to first token" and "tokens per second" are different performance metrics now that you understand this.

Modern · 03

RMSNorm & Pre-Norm — stabilising deep networks

When stacking 30, 80, or 120 layers deep, signal variance can explode or vanish. Normalisation acts as a guardrail at every layer.

Post-Norm (original 2017)
Normalise after the sublayer. The residual connection must pass through the norm before the next layer. At 30+ layers, gradients struggle to flow back cleanly — like passing a message through 30 translators, each distorting it slightly.
Pre-Norm (modern)
Normalise before the sublayer. The residual connection now carries raw, unadulterated signal from layer 1 to layer 120 — a highway express lane that bypasses all the toll booths. Dramatically stabilises ultra-deep training.

LayerNorm vs RMSNorm — the math

class RMSNorm(nn.Module):
    """Root Mean Square Layer Normalization — LLaMA, Mistral, Gemma."""
    def __init__(self, dim, eps=1e-6):
        super().__init__()
        self.eps    = eps
        self.weight = nn.Parameter(torch.ones(dim))   # learnable γ, no β
    def forward(self, x):
        rms = torch.sqrt(x.pow(2).mean(-1, keepdim=True) + self.eps)
        return (x / rms) * self.weight
The exam analogy

LayerNorm calculates the class average, subtracts it from every score, then divides by spread. RMSNorm just divides every score by the "overall magnitude." Both produce comparable rankings, but RMSNorm does it with fewer operations — crucial when grading billions of exams per second.

Modern · 04

SwiGLU — learned gating replaces blunt ReLU

Classic FFN used ReLU(x) = max(0, x). Any negative input: killed to zero. Brutally blunt — like a bouncer who either lets you in 100% or throws you out entirely.

The ReLU problem at scale

Input [-2.1, 0.5, -0.3, 1.8, -4.0] → output [0.0, 0.5, 0.0, 1.8, 0.0]. 60% of information destroyed every forward pass. "Dying ReLU": neurons with consistently negative outputs get gradient = 0 permanently and never learn again.

SwiGLU (Swish Gated Linear Unit) uses two parallel paths:

Path A — The Gate
SiLU(x) = x · sigmoid(x) — smooth value from −0.3 to +∞. Acts as a probability dial for how much signal should pass.
Path B — The Signal
A separate linear transformation carries the raw information.
The Merge
Gate × Signal element-wise. The gate controls how much of each feature dimension flows forward — the network learns to gate, rather than using a fixed threshold.
class SwiGLU_FFN(nn.Module):
    """Modern FFN — LLaMA, Mistral, etc."""
    def __init__(self, dim, hidden_dim):
        super().__init__()
        self.w_gate = nn.Linear(dim, hidden_dim, bias=False)   # gate path
        self.w_up   = nn.Linear(dim, hidden_dim, bias=False)   # signal path
        self.w_down = nn.Linear(hidden_dim, dim, bias=False)   # project back
    def forward(self, x):
        gate   = F.silu(self.w_gate(x))   # Path A
        signal = self.w_up(x)              # Path B
        return self.w_down(gate * signal)  # merge → project
Why SwiGLU wins at scale

No dead neurons: smooth curve means gradients always flow, even for negative inputs. Learned gating: network dynamically decides how much of each feature to pass based on context — ReLU can't do this. Trade-off: 3 weight matrices instead of 2; hidden dim is typically reduced to ²⁄₃ × 4d to keep parameter count similar.

The clock analogy

Each Q/K vector is a clock hand. Token at position 3 → rotated to 3 o'clock. Token at position 5 → rotated to 5 o'clock. Angle between them = 2 hours = relative distance. RoPE rotation: x'₂ᵢ = x₂ᵢcos(mθᵢ) − x₂ᵢ₊₁sin(mθᵢ)

Modern · 06

MHA → GQA — solving the KV cache memory wall

During autoregressive generation, the model must attend to all previous tokens. Recomputing K and V for every past token at every step is computationally impossible — so they're cached. This is the KV cache.

KV Cache Size = 2 × H × S_past × d_head × bytes_per_element32-layer model · 32 heads · 8K context · d_head=128 · float16 → 128 MB per layer → 4 GB total per user session
The memory wall

4 GB just to store KV cache for one user at 8K context. For 128K context windows: ~64 GB — exceeding most GPUs before the model even starts thinking. Standard MHA is the culprit: every head has its own K and V.

The spectrum: MHA → GQA → MQA

MethodQ headsK/V headsKV cache reductionQuality
MHA (Multi-Head)HH1× (baseline)Best quality
GQA (Grouped-Query)HG (1 < G < H)H/G reductionNear-MHA ✅
MQA (Multi-Query)H1H× reductionMeasurable drop ⚠

GQA is the sweet spot. Set G=H → GQA becomes MHA. Set G=1 → GQA becomes MQA. LLaMA 2 70B, Mistral 7B, and Gemma all use GQA.

GQA Step by Step
1. Project input X into Q, K, V. 2. Split Q into H query heads, group into G groups (H/G heads per group). 3. K and V have only G heads (one K/V pair per group). 4. Within each group: all H/G query heads attend using the same shared K and V. 5. Concatenate all head outputs → final linear projection.
Real-world result

GQA achieves 4–8× KV cache reduction with near-zero quality degradation. Gains come from reduced memory bandwidth and improved cache locality on GPU hardware. Uptraining shortcut: you don't need to train from scratch — average (mean-pool) the K and V weights within each group, then finetune for just 5–10% of original compute to recover quality.

DeepSeek's MLA — pushing the wall further

Multi-Head Latent Attention (MLA)

While GQA is the 2024 standard, DeepSeek-V3 uses low-rank compression to compress the entire KV cache into a tiny "latent vector," only decompressing inside the attention head. Result: up to 90% KV cache reduction vs standard MHA — enabling millions of tokens with minimal VRAM.

RMSNorm — implementation

Intuition

LayerNorm asks "how far is each value from the average?" RMSNorm asks "how big are the values overall?" Research found the mean-centering step wasn't contributing much — removing it makes the math simpler with no quality loss.

class RMSNorm(nn.Module):
    def __init__(self, dim, eps=1e-6):
        super().__init__()
        self.eps    = eps
        self.weight = nn.Parameter(torch.ones(dim))   # γ, init to 1
    def forward(self, x):
        rms = torch.sqrt(x.pow(2).mean(dim=-1, keepdim=True) + self.eps)
        return (x / rms) * self.weight
x.pow(2)→Square every element
.mean(dim=-1, keepdim=True)→Average across feature dim — mean of squares
+ eps → sqrt→Prevent div-by-zero → root-mean-square
x / rms * weight→Normalise then scale by learned γ per dimension
Pre-norm placement (used here, and in LLaMA/GPT-3)

x → Norm → Sublayer → + x. Residual connection is outside the normalisation — gradients flow through the skip path completely unimpeded, enabling stable training of 100+ layer models without careful LR tuning.

Impl · 03

RoPE — precompute frequencies & apply rotation

Three generations of positional encoding: (1) Sinusoidal — fixed absolute; (2) Learned embeddings — still absolute, limited to training length; (3) RoPE — rotation encodes relative distance, extrapolates beyond training length.

Step 1 — precompute cos/sin tables (done once, cached)

def precompute_rope_freqs(head_dim, max_seq_len, base=10000.0):
    # θᵢ = base^(-2i/d) — different frequency per dimension pair
    freqs     = 1.0 / (base ** (torch.arange(0, head_dim, 2).float() / head_dim))
    positions = torch.arange(max_seq_len).float()
    angles    = torch.outer(positions, freqs)   # [max_seq_len, head_dim//2]
    return torch.cos(angles), torch.sin(angles)
# Tables shape: [256, 16] — computed once, reused every step

Step 2 — apply rotation to Q and K (never V)

def apply_rope(x, cos, sin):
    seq_len = x.shape[2]
    cos = cos[:seq_len].unsqueeze(0).unsqueeze(0)   # [1, 1, seq, hd//2]
    sin = sin[:seq_len].unsqueeze(0).unsqueeze(0)
    x1  = x[..., ::2]    # even dims  (x_{2i})
    x2  = x[..., 1::2]   # odd dims   (x_{2i+1})
    out1 = x1 * cos - x2 * sin   # rotated even
    out2 = x1 * sin + x2 * cos   # rotated odd
    return torch.stack([out1, out2], dim=-1).flatten(-2)
# stack+flatten interleaves even/odd results back together
# Applied to Q and K only — V carries content, shouldn't be position-dependent
Dimension pairFrequency θRotation speedSensitivity
Low dims (pair 0,1)HighFast oscillationShort-range patterns (adjacent chars)
Mid dimsMediumModerateMedium-range patterns
High dims (pair d/2−1)LowSlow oscillationLong-range patterns (distant context)
Clock hands analogy

RoPE is a set of clock hands running at different speeds — the second hand captures fine timing (adjacent chars), minute hand captures medium patterns, hour hand captures broad structure. Together they encode position at every resolution simultaneously.

Impl · 04

GQA — grouped query attention & repeat_kv

This model: 8 query heads, 2 KV heads. Each KV head is shared by 4 query heads (n_rep = 4). K and V projections are 4× smaller than Q.

MHA analogy
8 students, each with their own personal copy of a textbook. Full redundancy. Full VRAM cost.
GQA analogy
4 students share one textbook copy. Each writes their own essay (query). 4× savings on books. Minimal quality loss.
# Projection sizes — K and V are 4× smaller than Q
self.q_proj = nn.Linear(d_model, n_heads    * head_dim, bias=False)  # 256→256
self.k_proj = nn.Linear(d_model, n_kv_heads * head_dim, bias=False)  # 256→64
self.v_proj = nn.Linear(d_model, n_kv_heads * head_dim, bias=False)  # 256→64

repeat_kv — stretch 2 KV heads to match 8 Q heads

def repeat_kv(x, n_rep):
    if n_rep == 1: return x          # MHA case — no repetition needed
    b, n_kv, seq, hd = x.shape
    return (x[:, :, None, :, :]      # insert repetition dim
              .expand(b, n_kv, n_rep, seq, hd)  # broadcast (no memory copy!)
              .reshape(b, n_kv * n_rep, seq, hd)) # merge into head dim
[b, 2, seq, 32]→Before: 2 KV heads
[b, 2, 1, seq, 32]→Insert repetition dim
[b, 2, 4, seq, 32]→.expand — repeat each KV head 4× (zero-copy view!)
[b, 8, seq, 32]→.reshape — now matches 8 Q heads ✅
expand is zero-copy

The 4 repetitions of each KV head are just different views of the same memory. GQA is nearly free at runtime — the savings come entirely from reduced KV cache size, not increased compute.

Full attention forward pass

# Project → Q [b,8,seq,32]   K [b,2,seq,32]   V [b,2,seq,32]
q = self.q_proj(x).view(b, seq, 8, 32).transpose(1,2)
k = self.k_proj(x).view(b, seq, 2, 32).transpose(1,2)
v = self.v_proj(x).view(b, seq, 2, 32).transpose(1,2)
# Apply RoPE to Q and K (not V!)
q = apply_rope(q, rope_cos, rope_sin)
k = apply_rope(k, rope_cos, rope_sin)
# Expand KV heads → [b,8,seq,32]
k = repeat_kv(k, 4); v = repeat_kv(v, 4)
# Scaled dot-product attention with causal mask
scores  = (q @ k.transpose(-2,-1)) * (1.0 / math.sqrt(32))
scores  = scores.masked_fill(causal_mask, float("-inf"))
weights = F.softmax(scores, dim=-1)
weights = F.dropout(weights, p=DROPOUT, training=self.training)
out     = weights @ v
# Merge heads → [b, seq, 256] then output projection
out = out.transpose(1,2).contiguous().view(b, seq, 256)
return self.o_proj(out)
The causal mask — preventing time travel

Upper-triangle positions get score = −∞ → becomes 0 after softmax. Each position can only attend to past tokens and itself. Without this mask the model "cheats" by peeking at future tokens during training — then fails at generation time where they don't exist.

SwiGLU FFN — implementation

SwiGLU(x) = W_down ( SiLU(W_gate · x) ⊙ (W_up · x) )SiLU(x) = x · σ(x) — smooth, non-monotonic, no dead neurons, gradient at x=0 is 0.5
ActivationFormulaDead neurons?Gradient at x=0
ReLUmax(0,x)Yes — permanentlyUndefined (0 or 1)
GELUx·Φ(x)No~0.5
SiLU/Swishx·σ(x)No0.5
class SwiGLU(nn.Module):
    def __init__(self, d_model, hidden_dim):
        super().__init__()
        self.w_gate = nn.Linear(d_model, hidden_dim, bias=False)
        self.w_up   = nn.Linear(d_model, hidden_dim, bias=False)
        self.w_down = nn.Linear(hidden_dim, d_model, bias=False)
    def forward(self, x):
        gate = F.silu(self.w_gate(x))   # gate: what to let through
        up   = self.w_up(x)              # up: the actual content
        return F.dropout(self.w_down(gate * up), p=DROPOUT, training=self.training)
# Dimensions: 256 → 680 (gate & up separately) → element-wise × → 256
# hidden_dim = 680 ≈ (2/3) × 1024 — keeps param count similar to 2-matrix ReLU FFN
Valve analogy

Classic FFN is a pipe — everything flows through the same path. SwiGLU is a valve system: one pipe carries the content, another pipe controls the valve deciding how much of each feature gets through. This learned "feature gating" lets the network selectively amplify or suppress different dimensions.

Model assembly — TransformerBlock & MiniLLM

class TransformerBlock(nn.Module):
    def __init__(self, d_model, n_heads, n_kv_heads, ffn_hidden_dim):
        super().__init__()
        self.attn_norm = RMSNorm(d_model)
        self.attention = GroupedQueryAttention(d_model, n_heads, n_kv_heads)
        self.ffn_norm  = RMSNorm(d_model)
        self.ffn       = SwiGLU(d_model, ffn_hidden_dim)
    def forward(self, x, rope_cos, rope_sin):
        x = x + self.attention(self.attn_norm(x), rope_cos, rope_sin)
        x = x + self.ffn(self.ffn_norm(x))
        return x

class MiniLLM(nn.Module):
    def __init__(self, ...):
        self.token_emb  = nn.Embedding(vocab_size, d_model)
        self.layers     = nn.ModuleList([TransformerBlock(...) for _ in range(n_layers)])
        self.final_norm = RMSNorm(d_model)
        self.lm_head    = nn.Linear(d_model, vocab_size, bias=False)
        self.lm_head.weight = self.token_emb.weight   # ← weight tying!
Weight tying

The embedding layer (token IDs → vectors) and lm_head (vectors → logits) are inverse operations — sharing the same weight matrix forces them to learn a consistent token space. Saves 65 × 256 = 16,640 parameters and acts as a regulariser. Like using the same dictionary to encode and decode a secret language.

Step 5 — gradient clipping

Generation — autoregressive sampling & temperature

@torch.no_grad()
def generate(model, prompt, max_new_tokens=500, temperature=0.8):
    model.eval()
    tokens = torch.tensor(encode(prompt), dtype=torch.long).unsqueeze(0)
    for _ in range(max_new_tokens):
        context   = tokens[:, -config["max_seq_len"]:]    # sliding window
        logits, _ = model(context)
        logits    = logits[:, -1, :] / temperature         # last position only
        probs     = F.softmax(logits, dim=-1)
        next_tok  = torch.multinomial(probs, num_samples=1)
        tokens    = torch.cat([tokens, next_tok], dim=1)
    return decode(tokens[0].tolist())

What's missing from this generator (and why it matters)

KV cache: recomputing K/V each step makes generation O(n²) — storing them makes it O(n). Top-k / top-p sampling: cuts off the garbage tail of the probability distribution. Both are standard additions once the baseline model is verified.

RAG · 04

Ingestion Part 2 — Chunking strategies

Once parsed, a document is too large to embed as a single vector. It must be split into chunks. This is where 80% of RAG systems fail.

Every cut destroys context

The art of chunking is choosing where to cut — at logical boundaries, not arbitrary character counts.

Strategy 1 — Fixed-size chunking (the naive approach)

Split every N tokens with overlap. Ignores all boundaries. Faithfulness Score: ~0.47 💔 — often cuts mid-sentence or mid-thought.

def chunk_fixed(text: str, chunk_size: int = 200, overlap: int = 50) -> List[str]:
    words = text.split()
    chunks = []
    for i in range(0, len(words), chunk_size - overlap):
        chunk = " ".join(words[i:i + chunk_size])
        if chunk.strip(): chunks.append(chunk.strip())
    return chunks

Strategy 2 — Recursive chunking (the standard)

Splits logically: paragraphs → sentences → characters. Respects natural boundaries. Faithfulness Score: ~0.80 ✅

def chunk_recursive(text: str, max_size: int = 200) -> List[str]:
    paragraphs = [p.strip() for p in text.split("

") if p.strip()]
    chunks = []
    for para in paragraphs:
        words = para.split()
        if len(words) <= max_size:
            chunks.append(para)
        else:
            sentences = para.replace(". ", ".
").split("
")
            current, current_len = [], 0
            for sent in sentences:
                sent_len = len(sent.split())
                if current_len + sent_len > max_size and current:
                    chunks.append(" ".join(current))
                    current, current_len = [sent], sent_len
                else:
                    current.append(sent); current_len += sent_len
            if current: chunks.append(" ".join(current))
    return [c for c in chunks if len(c.split()) > 10]
Always prefer recursive

Fixed cuts mid-thought. Recursive respects paragraph boundaries. Use fixed-size only when you have a specific reason.

Reciprocal Rank Fusion (RRF)

Fuses two ranked lists into one. Elegant — no hyperparameters to tune.

def reciprocal_rank_fusion(semantic, keyword, k=60):
    scores = {}
    for rank, (idx, _) in enumerate(semantic):
        scores[idx] = scores.get(idx, 0) + 1 / (k + rank + 1)
    for rank, (idx, _) in enumerate(keyword):
        scores[idx] = scores.get(idx, 0) + 1 / (k + rank + 1)
    return sorted(scores.items(), key=lambda x: x[1], reverse=True)

⚡ Quick Recall — All 9 Weeks
🔢 Part 1 — Maths
  • Dot product = element-wise multiply then sum. High = similar direction
  • Matrix multiply: [M×K] @ [K×N] = [M×N]. Inner dims must match
  • Chain rule: d(f∘g)/dx = f′(g(x))·g′(x). This IS backprop
  • Gradient descent: w ← w − lr·∂L/∂w. Move opposite to gradient
  • Cross-entropy: −log(p_correct). Perfect→0. Random over 65 tokens→4.17
  • Softmax: eˣⁱ / Σeˣʲ. Turns logits into probabilities that sum to 1
🧠 Part 2 — Neural Nets
  • Perceptron: ŷ = f(w·x + b). Dot product + bias + activation
  • XOR needs hidden layer — not linearly separable with one boundary
  • Backprop = chain rule applied backwards through the graph
  • ReLU dead neuron: if output always negative, gradient=0, never learns
  • Training loop: forward → loss → zero_grad → backward → step
  • Random init essential — zero init → all neurons identical, symmetry never breaks
📝 Part 3 — Text → Attention
  • BPE: subword tokenisation. "strawberry"→["st","raw","berry"] — can't count 'r'
  • Embedding = 768-number address in meaning-space. Similar words are near
  • Positional encoding injects order — attention is permutation-invariant without it
  • Attention: softmax(QKᵀ/√d_k)·V. Q=query, K=key, V=value
  • Scale by √d_k: prevents softmax saturation when dot products are large
  • KV cache: K and V don't change once computed. Cache and reuse; only Q is new
🔥 Part 4 — PyTorch
  • Tensor = n-dim array. Shape is metadata over a flat memory block
  • .view() is zero-copy (strides). .permute() → non-contiguous → .reshape() needed
  • Autograd: requires_grad=True → PyTorch records every op → .backward() computes all grads
  • PyTorch accumulates gradients — zero_grad() before every backward()
  • Broadcasting: compare shapes right-to-left. Sizes must be equal or one must be 1
  • Debug trinity: print(x.shape, x.dtype, x.device) — always first
⚙ Part 6 — Deep Learning Engine
  • All DL is matmul: neurons=input@W.T+b, attention=softmax(Q@K.T)@V, embeddings=one_hot@W_emb
  • GPU: 5000+ simple cores vs CPU's 16 powerful cores. Matmul is embarrassingly parallel → 1800× faster
  • float16: 2 bytes vs 4 (float32). Halves VRAM. Risk: overflow to NaN
  • Without ReLU/activation: stacking N Linear layers = one Linear layer
  • Training: forward→loss→zero_grad→backward→clip→step
🚀 Part 7 — Modern LLM (W6)
  • RMSNorm: remove mean subtraction and β. Just x/RMS(x)·γ. Same stability, less compute
  • Pre-norm: normalise BEFORE sublayer → residual carries raw signal → stable at 100+ layers
  • SwiGLU: gate path (SiLU) × signal path (Linear). Learned gating, no dead neurons
  • RoPE: rotate Q,K by position → Q·K depends only on relative distance (m−n)
  • GQA: share K,V across groups of Q heads → 4–8× KV cache reduction
  • Modern block order: RoPE→RMSNorm→GQA→residual→RMSNorm→SwiGLU→residual
🔧 Part 8 — MiniLLM (W7)
  • Char-level tokeniser: 65 tokens, trivial setup — focus on architecture not vocab
  • y = x shifted right by 1. L predictions per sequence for free
  • repeat_kv: .expand() is zero-copy. 2 KV heads → 8 Q heads with no memory alloc
  • Weight tying: lm_head.weight = token_emb.weight. Saves 16K params, regularises
  • Step-0 loss ≈ ln(vocab_size) = 4.17. If wildly off → bug
  • Gradient clipping: scale down all grads if norm > 1.0. Safety valve for bad batches
🤖 Part 9 — Agents (W8)
  • Agent = LLM + Tools + Loop. LLM has NO memory — re-send full history every call
  • LLM doesn't execute tools — it requests them via JSON. YOUR code runs them
  • ReAct: THOUGHT (plan) → ACTION (tool request) → OBSERVATION (result) → repeat
  • Context engineering problem: prompt grows every iteration. 266→355→432→507…
  • Native function calling > string parsing in prod. No regex fragility
  • max_iterations is NOT optional. Hallucinating agent = infinite loop + infinite cost
🗂 Part 10 — RAG (W9)
  • RAG = Retrieve relevant chunks → Augment prompt → Generate grounded answer
  • Recursive chunking (0.80) vs fixed-size (0.47): always split on natural boundaries
  • Contextual retrieval: LLM prepends context to every chunk before embedding. 67% fewer failures
  • Hybrid search: semantic catches meaning; BM25 catches exact dates/IDs/commands. RRF fuses both
  • Two-pass retrieval: Bi-Encoder (fast, rough top-50) → Cross-Encoder (slow, precise top-3)
  • U-shaped attention: place best chunks at the END of context, right before the question
